어떠한 카톡방에서 있었던 대화에 대한 정보가 필요할 수 있음. 
ex) 아 얘가 저번에 딥러닝 과제로 한다고 했던 주제가 뭐였더라...
ex) 부장님이 저번에 가입하라고 했던 홈페이지가 뭐였더라...
이때 AI가 해당 카톡방에서의 대화를 요약해서 보유하고 있다면? 
질의응답을 통해 원하는 답 겟또

이를 위해...
여러 텍스트들이 주어진다(길지 않겠지) -> 모델은 이를 계속 학습하고 요약 정보를 보유(꼭 요약 정보가 필요한 건 아닌 것 같기도?) -> 사용자가 질의 -> 이에 대한 응답

그러려면 대화 텍스트랑, 그에 대한 질문 및 답변 텍스트가 필요할듯?


기존 질의응답(QA) task의 데이터를 보면 그냥 평서문 형태로 지문이 주어지고 거기서 답을 찾는 형식임. 
ex) 양주시는 올해 뭐뭐를 시행하기로 하고...
근데 내가 하고 싶은 task의 데이터는 대화 형식임
ex) XX야 내일 어벤져스 보러 가자!
그러면 우선 대화 형식을 지문 형태로 변환한 다음, 거기에 대해서 질의응답 모델을 돌리면 되려나?

ex)
A: 배고프다 뭐먹지
B: 제육볶음 먹어
A: 좋다 먹으러 가야지
Qustion: A가 먹은 음식이 뭐야?
Answer: 제육볶음

VS

A는 배고프다고 말했다. B는 제육볶음 먹으라고 말했다. A는 좋다 먹으러 가야겠다고 말했다. 
Qustion: A가 먹은 음식이 뭐야?
Answer: 제육볶음

대화 텍스트를 그냥 집어넣은 후 질의를 한 것과, 대화 텍스트를 지문 형태로 변환한 것의 성능 차이를 비교하기?


1) 대화 텍스트를 얻는다(D)
2) 이를 지문 형태로 변환(D')
3) 기존 QA 모델을 활용하여(BERT 등), D를 넣었을 때와 D'를 넣었을 때 QA의 성능 비교
